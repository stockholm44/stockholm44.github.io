2.2 Outlier detection¶

위에서 데이터를 불러들였으면 원래는 하나하나 탐색을 해봐야하겠지만 이 노트북은 약간 좀 이미 여러번 탐색을 한 사용자에 대한 노트북이라서 미리 이상치를 제거하는 함수를 구현합니다.

일일이 아웃라이어의 행을 찾아서 없에도 될정도로 수가 작긴하지만 나중에도 이렇게 일일이 한다면 마치 아이폰으로 통화나 하는 꼴이겠죠? detect_ouliers라는 함수를 만들어서 아웃라이어의 행의 index를 저장하여 return 하겠습니다.

아웃라이어(이상치)는 예측에 많은 영향을 끼칩니다.

이 노트북에서는 Tukey method (Tukey JW., 1977) 사분할 영역에서 첫째, 셋째 사이의 구간을 IQR로 지정하고 이 구간 바깥영역을 아웃라이어로 간주합니다. 또한 n이라는 parameter를 받아서 n개의 feature에서 이상치가 된다면 그 때 이상치에 해당하는 행으로 추가합니다.

이번에 타이타닉데이터에서는 연속형 데이터(Age, SibSp, Sarch and Fare)에서 2개이상 아웃라이어 영역에 있으면 제외할 행으로 지정하기로 하겠습니다

# Outlier detection 

def detect_outliers(df,n,features):
    """
    Takes a dataframe df of features and returns a list of the indices
    corresponding to the observations containing more than n outliers according
    to the Tukey method.
    """
    outlier_indices = [] 
    
    # iterate over features(columns)
    for col in features:
        # 1st quartile (25%)
        Q1 = np.percentile(df[col], 25)
        # 3rd quartile (75%)
        Q3 = np.percentile(df[col],75)
        # Interquartile range (IQR)
        IQR = Q3 - Q1
        
        # outlier step
        outlier_step = 1.5 * IQR
        
        # Determine a list of indices of outliers for feature col
        outlier_list_col = df[(df[col] < Q1 - outlier_step) | (df[col] > Q3 + outlier_step )].index
        
        # append the found outlier indices for col to the list of outlier indices 
        outlier_indices.extend(outlier_list_col)
        
    # select observations containing more than 2 outliers
    outlier_indices = Counter(outlier_indices) 
    print(outlier_indices)
    multiple_outliers = list( k for k, v in outlier_indices.items() if v > n )
    
    return multiple_outliers

Fare feature는 매우 skew가 높은 분포를 보입니다. skew가 매우 크다는 것은 매우 불균형한 분포를 보이고 있다는 뜻이고 이렇게 불균일한 데이터 분포는 추후에 예측이 잘되지 않게 됩니다. 데이터에 아주 고액의 Fare가 존재하여 scale을 변경하여도 큰 영향을 끼칠 것으로 보인다. 그래서 Fare feature는 skewness를 줄이고 정규분포에 가깝게 하기위해 feature engineering이 필요합니다. 보통 2가지 방법을 많이 사용하는데 첫째는 log를 취하는거고 두번째는 box-cox라느 방법입니다. 여기서는 log로 정규분포로 변환이 될 수 있게 해보겠습니다.

map 메소드를 사용하면 한개 column에 동시에 함수를 적용할 수 있습니다. 간단한 +-/%는 구지 map이나 apply를 사용하지 않아도 되지만 조금복잡하게 되면 함수를 따로 만들어 map을 하거나 그것보다는 간단하면 map안에 lambda로 직접 넣어주셔도 됩니다.

dataset['Fare'] = dataset['Fare'].map(lambda i: np.log(i) if i > 0 else 0)

4. Filling missing Values¶

4.1 Age¶

전에 확인했든이 Age featrue는 256개 행이 값이 없습니다. 나이대에 따라 생존확률이 다르기 때문에 Age feature의 결측치는 살려두는 것이 좋을 듯합니다.

Age의 값을 추정하기 위해 Sex, Parch, Pclass, SibSp와의 상관관계를 확인해보겠습니다.

In [30]:
g = sns.factorplot(y='Age', x='Sex', data=dataset, kind='box')
g = sns.factorplot(y='Age', x='Sex', hue='Pclass', data=dataset, kind='box')
g = sns.factorplot(y='Age', x='Parch', data=dataset, kind='box')
g = sns.factorplot(y='Age', x='SibSp', data=dataset, kind='box')
  • Age의 분포는 남녀가 비슷해 보입니다. 그래서 성별은 그닥 나이예측에 도움이 될것 같진 않아 보이네요.
  • 그런데 1등석 승객이 2등석 승객보다 나이가 많고 3등석보다는 또 나이가 많습니다.
  • 게다가 Parch가 클수록 나이가 많고 SibSp가 많을수록 어린것으로 보입니다.
  • 그래서 Pclass, Parch, SibSP는 Age를 예측하는 중요한 정보로 생각이 듭니다.
In [31]:
# convert Sex into categorical value 0 for male and 1 for female
dataset['Sex'] = dataset["Sex"].map({'male':0, 'female':1})

g = sns.heatmap(dataset[['Age','Sex','SibSp','Parch','Pclass']].corr(), cmap="BrBG", annot=True, vmin=-1, vmax=1)

상관관계 Map도 아까의 결론과 비슷한 결과를 나타내고 있습니다. 성별에는 큰 상관성이 없고 Pclass와는 어느정도 음의 상관관계를, Parch, SibSp와도 약한 음의 상관관계를 가집니다.

그래서 이 세가지로 Age를 예측하여 채워넣겠습니다.

index_NaN_age에 Age가 NaN값인 index를 리스트로 저장해 놓습니다.

In [33]:
# Filling missing value of Age 

## Fill Age with the median age of similar rows according to Pclass, Parch and SibSp
# Index of NaN age rows
index_NaN_age = list(dataset['Age'][dataset["Age"].isnull()].index)

for문으로 i index의 SibSP, Parch Pclass가 다 같은 Age의 median을 할당하되 셋다 동시에 만족하는 것이 없으면 전체 Age의 중심값으로 채웁니다.

In [34]:
for i in index_NaN_age:
    age_med = dataset["Age"].median()
    age_pred = dataset["Age"][((dataset['SibSp'] == dataset.iloc[i]["SibSp"]) 
                               & (dataset['Parch'] == dataset.iloc[i]['Parch']) 
                               & (dataset['Pclass']== dataset.iloc[i]["Pclass"]))].median()
    if not np.isnan(age_pred):
        dataset['Age'].iloc[i] = age_pred
    else:
        dataset['Age'].iloc[i] = age_med

g = sns.factorplot(x='Survived',y='Age', data=train, kind='violin')

생존률에 따른 전체 중심치는 여전히 비슷해보이는 것 같쥬?. 그러나 나이대별 바이올린 플랏으로 보면 어린나이대의 생존이 여전히 높은 것을 볼수 있습니다.

def plot_learning_curve(estimator, title, X, y, ylim=None, cv=None, 
                        n_jobs=-1, train_sizes=np.linspace(.1,1.0,5)):
    plt.figure()
    plt.title(title)
    if ylim is not None:
        plt.ylim(*ylim)
    plt.xlabel("Training examples")
    plt.ylabel('Score')
    train_sizes, train_scores, test_scores = learning_curve(
        estimator, X, y, cv=cv, n_jobs=n_jobs, train_sizes=train_sizes)
    train_scores_mean = np.mean(train_scores, axis=1)
    train_scores_std = np.std(train_scores, axis=1)
    test_scores_mean = np.mean(test_scores, axis=1)
    test_scores_std = np.std(test_scores, axis=1)
    plt.grid()
    
    plt.fill_between(train_sizes, train_scores_mean - train_scores_std,
                    train_scores_mean + train_scores_std, alpha=0.1, color='r')
    plt.fill_between(train_sizes, test_scores_mean - test_scores_std,
                    test_scores_mean + test_scores_std, alpha=0.1, color='g')
    plt.plot(train_sizes, train_scores_mean, 'o-', color='r', label='Training Score')
    plt.plot(train_sizes, test_scores_mean, 'o-', color='g', label='Cross-validation score')

    plt.legend(loc='best')
    return plt
